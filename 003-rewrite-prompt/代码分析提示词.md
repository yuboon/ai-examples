# 代码分析提示词 - 用于从旧代码提取需求

> 复制以下提示词给AI，用于分析旧项目并生成需求文档

---

## 📌 提示词版本1：完整版（推荐）

```
你是一位资深的软件架构师和需求分析师。我现在需要你帮我分析一个旧项目的代码，
并提取出完整的需求文档，以便后续使用AI进行代码重写。

【任务目标】
1. 分析现有代码库，理解项目的业务逻辑和技术实现
2. 提取并整理成结构化的需求文档
3. 确保文档格式对AI代码生成友好

【分析步骤】

第一步：项目概览分析
- 扫描项目结构，识别项目类型（web/app/mobile/library等）
- 识别使用的技术栈（语言、框架、库）
- 理解项目的整体架构模式
- 估算项目规模（文件数、代码行数、模块数）

第二步：功能模块识别
- 分析路由/页面结构，识别所有功能页面/模块
- 分析组件/类结构，识别可复用单元
- 分析API/服务层，识别业务功能
- 整理成功能清单（F001、F002...）

第三步：数据模型提取
- 分析数据结构（interface/model/schema/entity等）
- 识别实体及其属性
- 识别实体间的关系（1:1、1:N、N:M）
- 绘制数据关系图

第四步：业务逻辑梳理
- 分析每个功能的核心业务流程
- 识别业务规则和约束条件
- 识别状态管理逻辑
- 识别异常处理逻辑

第五步：API/接口分析
- 提取所有API端点
- 分析请求/响应格式
- 识别认证授权机制
- 整理错误码定义

第六步：UI/交互分析
- 分析页面布局结构
- 识别UI组件及其props
- 分析用户交互流程
- 识别特殊状态（loading/error/empty）

第七步：配置与依赖分析
- 分析构建配置
- 识别环境变量
- 分析依赖关系
- 识别第三方服务集成

【输出要求】

请按照以下模板输出需求文档，确保：
1. 所有代码片段使用实际代码中的类型定义
2. 所有功能描述基于实际代码逻辑
3. 保留重要的业务规则和约束
4. 标注代码中的关键路径（行号/文件路径）
5. 识别技术债务和需要改进的地方

【输出格式】
输出为Markdown格式，使用以下模板（详见：AI项目重写-需求文档模板.md）：

```markdown
# [项目名称] 需求文档

## 1. 项目概述
[项目目标描述]
[核心功能清单]
[约束条件]

## 2. 架构设计
[系统架构说明]
[目录结构]
[技术选型]

## 3. 数据模型
[所有实体定义]
[数据关系图]

## 4. 功能详细规格
[每个功能的详细规格，使用4.1节的模板]

## 5. API规格（如适用）
[所有API详细定义]

## 6. 状态管理
[全局状态定义]
[状态流转]

## 7. UI设计规范
[组件规范]
[样式规范]

## 8. 安全要求
[认证授权]
[数据安全]

## 9. 测试要求
[测试类型]
[关键场景]

## 10. 技术债务与改进建议
[列出发现的技术债务]
[提出改进建议]

## 11. AI执行指令
[代码生成顺序]
[代码规范]
```

【注意事项】
1. 如果代码中有不清晰的逻辑，请标注出来并给出推测
2. 如果发现代码中的bug或潜在问题，请在技术债务部分记录
3. 保持客观，不要添加原文档中没有的功能
4. 对于缺失的文档化内容，基于代码进行合理推断
5. 引用关键代码位置时使用格式：文件路径:行号

现在请开始分析，项目代码路径为：[你的项目路径]

## 📌 提示词版本2：精简版

```
请帮我分析这个旧项目的代码并提取需求文档。

项目代码路径：[你的项目路径]

请分析并输出：
1. 项目概述（目标、核心功能、技术栈）
2. 数据模型（所有实体定义和关系）
3. 功能清单（每个功能的用户故事和业务规则）
4. API规格（所有接口定义）
5. 目录结构和架构说明
6. 技术债务和改进建议

输出为Markdown格式，确保AI可以根据此文档重新生成代码。
```

## 📌 提示词版本3：分步执行版

### 步骤1：项目扫描
```
请扫描项目 [项目路径] 并输出：
1. 项目类型和技术栈
2. 目录结构树
3. 主要模块和文件列表
4. 依赖包列表
```

### 步骤2：功能识别
```
基于上面的扫描结果，请分析并输出：
1. 所有页面/路由列表
2. 所有组件列表
3. 所有API/服务列表
4. 核心业务流程描述
```

### 步骤3：数据模型提取
```
请提取项目中的所有数据模型：
1. 实体定义（interface/type/model）
2. 实体属性和类型
3. 实体间关系
4. 数据流转说明
```

### 步骤4：功能规格整理
```
请为每个功能编写详细规格：
1. 用户故事
2. 输入输出定义
3. 业务规则
4. 异常处理
5. 验收标准
```

### 步骤5：生成完整文档
```
请整合以上分析，生成完整的需求文档（使用提供的模板）
```

3. 业务逻辑流程
4. 输出结果定义

```

```

## 🎯 针对不同项目类型的定制提示词

```

1. 路由结构和页面层级

3. 状态管理方案
4. API调用封装
5. 样式组织方式
6. 构建配置
```

```
请重点分析：
1. API端点列表和路由
2. 中间件和拦截器
3. 数据库模型和关系
4. 认证授权机制
5. 错误处理策略
6. 数据验证规则
```

### 移动应用项目
```
请重点分析：
1. 页面/屏幕导航流程
2. 本地数据存储
3. 网络请求封装
4. 设备能力调用（相机、定位等）
5. 原生模块集成
6. 性能优化措施
```
